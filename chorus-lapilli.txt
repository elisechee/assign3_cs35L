The main difference here from the tic tac toe implementation is that Chorus Lapilli has a "moving phase" that occurs after each player has reached 3 moves where they can no longer place new pieces, and must instead move an existing piece to an adjacent sqaure each subsequent turn. And additionally, theres a rule that forces a player who occupies the center square to either win or vacate it. 

I introduced new state variables that counted how many pieces each player had placed and a boolean variable that becomes true when the Chorus Lapilli phase of the game starts. 

    const [numMovesX, setNumMovesX] = useState(0);
    const [numMovesO, setNumMovesO] = useState(0);

    const isMovingPhase = numMovesX >= 3 && numMovesO >= 3; // both players done placing

The way to check if the second click is a valid move or not while accounting for edge, corner, and center piece moves. The it was done is using nested for loops to basically traverses the board as a flat array and then only include the ones that are within the board. and we traverse with indices that have row by dividing pieceToMove/3 and col with mod 3 for pieceToMove. This gives corner pieces 3 valid neighbors, edge pieces 5, and center piece 8.

    const validMoves = [];
      const row = Math.floor(pieceToMove / 3);
      const col = pieceToMove % 3;
      
      for (let r = row - 1; r <= row + 1; r++) {
        for (let c = col - 1; c <= col + 1; c++) {
          if (r >= 0 && r < 3 && c >= 0 && c < 3) {
            validMoves.push(r * 3 + c);
          }
        }
      }

On case of it being an invalid move then it gets rejected, and a timed state flag provides feadback, and resets the move variables isValidMove and isFirstClick and sets isValidMove to false.
